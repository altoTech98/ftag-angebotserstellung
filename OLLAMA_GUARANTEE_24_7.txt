     - Überwacht Ollama alle 30 Sekunden

     - Startet Ollama beim Login automatisch

  3. OLLAMA NATIVE RECOVERY
     - Ollama eigenes Error-Handling
     - Datenbank-Recovery bei Crashes

===============================================================================
QUICK START
===============================================================================

Option A - Automatisch (empfohlen):
  1. Klick auf: START_24_7_MODE.bat (als Administrator)
  2. Script installiert Windows Task Scheduler
  3. Backend startet automatisch
  4. Fertig!

Option B - Manuell:
  1. PowerShell als Administrator:
     PS> .\setup_ollama_autostart.ps1
  
  2. Terminal in backend/ dir:
     $ python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000

  3. Check Status:
     PS> .\CHECK_OLLAMA_STATUS.ps1

  self.health_check_interval = 30           # Sekunden zwischen Checks
  self.max_restart_attempts = 10            # Max Auto-Restarts
  self.failure_backoff = 5                  # Backoff-Basis (exponentiell)

===============================================================================
FEHLERBEHANDLUNG
===============================================================================

Problem: Ollama läuft nicht
  → Watchdog startet es automatisch
  → Check: .\CHECK_OLLAMA_STATUS.ps1
  → Logs: logs/app.log

Problem: Scheduled Task funktioniert nicht
  → Muss als Administrator laufen!
  → Neuanlage: .\setup_ollama_autostart.ps1

Problem: Binary nicht gefunden
  → Fallback auf "ollama" Kommando (PATH)
  → Oder: Ollama neu installieren

===============================================================================
DEINSTALLATION (Falls nötig)
===============================================================================

PowerShell als Admin:
  PS> Unregister-ScheduledTask -TaskName "Ollama_24_7_Watchdog" -Confirm:$false

Backend Watchdog wird automatisch deaktiviert mit Backend.
Ollama bleibt aber laufen (separat konfiguriert).
