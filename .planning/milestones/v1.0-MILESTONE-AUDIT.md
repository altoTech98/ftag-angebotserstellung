## Cross-Phase Integration Issues

### CRITICAL: Frontend V2 Result Shape Mismatch

`AnalysePage.jsx:runFolderWorkflow` reads `result.requirements.positionen` and `result.matching.summary` after v2 analysis completes. The v2 pipeline response has completely different keys: `positionen`, `match_results`, `adversarial_results`, `gap_results`, `analysis_id`.

**Fix:** Update `runFolderWorkflow` lines 496-503 to read v2-shaped response keys.

### CRITICAL: Frontend Calls V1 Offer Endpoint After V2 Analysis

After v2 analysis, `runFolderWorkflow` calls `api.generateResult(result.requirements, result.matching)` which posts to `/api/result/generate` (v1). The v2 endpoint `POST /api/offer/generate` expects `{"analysis_id": "..."}`.

**Fix:** Add `api.generateV2Offer(analysisId)` function in api.js; update `runFolderWorkflow` to call it with `result.analysis_id`.

### CRITICAL: V2 Feedback Loop Severed

`CorrectionModal.jsx` sends corrections to `/api/feedback` (v1 store). The v2 AI matcher reads from `FeedbackStoreV2` (v2 store). No corrections from the UI ever reach the v2 matching pipeline.

**Fix:** Add `api.saveV2Feedback()` function; detect v2 analysis context in CorrectionModal and call v2 endpoint.
