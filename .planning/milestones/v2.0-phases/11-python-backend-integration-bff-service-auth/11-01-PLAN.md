<objective>
Replace Python's existing JWT auth middleware with service key authentication, add SSE token validation, wire SSE token enforcement into the stream endpoint, and update CORS configuration for the Next.js frontend origin.

Purpose: Python backend must only accept requests from the Next.js BFF proxy (via shared service key) rather than directly from browsers. SSE streaming endpoints use short-lived HMAC tokens for direct browser connections. The SSE token must be validated end-to-end: issued by Next.js (Plan 02) and enforced by the Python stream endpoint (this plan). This is the foundation that Plan 02 (Next.js BFF proxy) builds on.

Output: service_auth.py, sse_token_validator.py, updated config.py, main.py, and analyze.py, passing tests.
</objective>
