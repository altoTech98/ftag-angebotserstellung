  <name>Task 1: Admin server actions (replace stubs with real implementations)</name>
  <files>frontend/src/lib/actions/admin-actions.ts</files>
  <action>
Replace the stub implementations from 15-00 with full server actions. All actions must:
- Check session via `auth.api.getSession({ headers: await headers() })`
- Check permission via `auth.api.userHasPermission` with appropriate admin permission
- Call logAuditEvent after successful mutations (fire-and-forget, no await to avoid blocking)

Implement these server actions:

1. `listUsers(search?: string, offset = 0, limit = 50)`:
   - Permission: admin:manage-users
   - Call `auth.api.listUsers({ query: { searchValue: search, searchField: 'name', limit, offset } })`
   - IMPORTANT: Response is `{ users, total }` not a flat array (see Pitfall 1)
   - Return `{ users, total }`

2. `inviteUser(formData: FormData)`:
   - Permission: admin:manage-users
   - Extract name, email, role from FormData
   - Call `auth.api.createUser({ body: { email, password: crypto.randomUUID(), name, role } })`
   - IMPORTANT: Use `body:` parameter for server-side calls (see Pitfall 6)
   - Log audit event: action='user_invited', details=`${name} (${email}) als ${role} eingeladen`
   - TODO: Send invitation email (wired in Plan 15-03)
   - revalidatePath('/admin')
   - Return new user

3. `updateUserRole(userId: string, role: string)`:
   - Permission: admin:manage-users
   - Call `auth.api.setRole({ body: { userId, role } })`
   - Log audit: action='role_changed', details=`Rolle geaendert zu ${role}`, targetId=userId, targetType='user'
   - revalidatePath('/admin')

4. `toggleUserBan(userId: string, ban: boolean, reason?: string)`:
   - Permission: admin:manage-users
   - If ban: `auth.api.banUser({ body: { userId, banReason: reason } })`
   - If unban: `auth.api.unbanUser({ body: { userId } })`
   - Log audit: action=ban?'user_deactivated':'user_activated'
   - revalidatePath('/admin')

5. `getSystemSettings()`:
   - Permission: admin:manage-settings
   - `prisma.systemSettings.findUnique({ where: { id: 'default' } })`
   - If null, return defaults object (not null)
   - Mask claudeApiKey: show only last 4 chars if present

6. `updateAnalyseSettings(formData: FormData)`:
   - Permission: admin:manage-settings
   - Extract defaultConfidence (Float), maxUploadSizeMB (Int), validationPasses (Int)
   - Upsert SystemSettings with these values + updatedBy = session.user.id
   - Log audit: action='settings_changed', details='Analyse-Einstellungen aktualisiert'
   - revalidatePath('/admin')

7. `updateSecuritySettings(formData: FormData)`:
   - Permission: admin:manage-settings
   - Extract sessionTimeoutMin (Int)
   - Upsert SystemSettings
   - Log audit: action='settings_changed', details='Sicherheits-Einstellungen aktualisiert'
   - revalidatePath('/admin')

8. `updateApiKeySettings(formData: FormData)`:
   - Permission: admin:manage-settings
   - Extract claudeApiKey (String)
   - Upsert SystemSettings (store as-is for now; encryption is a deployment concern)
   - Log audit: action='api_key_changed', details='API-Schluessel aktualisiert'
   - revalidatePath('/admin')
