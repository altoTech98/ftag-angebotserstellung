  <action>
1. Add to `frontend/src/lib/actions/project-actions.ts` (append, do not replace existing actions):

   `shareProject(projectId: string, email: string, role: string = "viewer")`:
   - Authenticate + check project:share permission
   - Verify caller is owner or admin of the project
   - Look up target user by email: `prisma.user.findUnique({ where: { email } })`
   - Return error object { error: "Benutzer nicht gefunden" } if user not found
   - Return error if sharing with self
   - Check for existing share: `prisma.projectShare.findUnique({ where: { projectId_userId: { projectId, userId } } })`
   - Return error { error: "Projekt bereits geteilt" } if exists
   - Create share: `prisma.projectShare.create({ data: { projectId, userId: targetUser.id, role } })`
   - revalidatePath for project detail page
   - Return { success: true, share }

   `removeShare(shareId: string)`:
   - Authenticate
   - Find share, verify caller is project owner or admin
   - Delete: `prisma.projectShare.delete({ where: { id: shareId } })`
   - revalidatePath

   `getProjectShares(projectId: string)`:
   - Authenticate
   - Return: `prisma.projectShare.findMany({ where: { projectId }, include: { user: { select: { id: true, name: true, email: true } } } })`

2. Create `frontend/src/components/projects/share-dialog.tsx` (client component):
   - Props: projectId, projectName, open, onOpenChange
   - On mount/open: fetch current shares via getProjectShares server action
   - Form: Email input (type="email") + role select (Betrachter/Bearbeiter) + "Teilen" button
   - On submit: call shareProject, show success toast or error toast
   - Current shares list: each row shows user name, email, role badge, remove button (X icon)
   - Remove button calls removeShare, refreshes list
   - Use Dialog, Input, Button, Label from shadcn/ui
   - Loading states with useTransition for share/remove actions

3. Create test file with behavior tests. Mock prisma for user lookup, share creation, permission checks.
