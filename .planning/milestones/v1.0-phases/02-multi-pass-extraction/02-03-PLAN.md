  <name>Task 1: Pass 2 semantic + Pass 3 validation + pipeline orchestrator</name>
  <files>backend/v2/extraction/pass2_semantic.py, backend/v2/extraction/pass3_validation.py, backend/v2/extraction/pipeline.py, backend/v2/extraction/__init__.py, backend/tests/test_v2_pipeline.py</files>
  <action>
    Create `backend/v2/extraction/pass2_semantic.py`:
    - `async def extract_semantic(parse_result: ParseResult, existing_positions: list[ExtractedDoorPosition] | None = None) -> list[ExtractedDoorPosition]`
    - Use `anthropic.AsyncAnthropic()` client. If AsyncAnthropic doesn't support messages.parse(), fall back to `asyncio.to_thread()` wrapping sync `Anthropic().messages.parse()`.
    - Chunk the document using `chunk_by_pages(parse_result.text, parse_result.page_count)`.
    - For each chunk: build prompt from PASS2_USER_TEMPLATE with chunk text and existing_positions JSON. Call `client.messages.parse(model="claude-opus-4-6", max_tokens=16384, system=PASS2_SYSTEM_PROMPT, messages=[...], output_format=ExtractionResult)`.
    - 3x retry with exponential backoff (2^attempt seconds). On total failure: log warning, skip chunk, continue.
    - Check `response.stop_reason` -- if "max_tokens", log warning about possible truncation.
    - Tag all extracted fields with FieldSource(dokument=parse_result.source_file, konfidenz=0.9, seite=chunk.start_page).
    - Return all positions from all chunks combined.

    Create `backend/v2/extraction/pass3_validation.py`:
    - `async def validate_and_enrich(positions: list[ExtractedDoorPosition], original_texts: list[str], source_files: list[str]) -> list[ExtractedDoorPosition]`
    - Build prompt from PASS3_USER_TEMPLATE with position summaries (positions_nr + key fields only, NOT full JSON to avoid context overflow per Pitfall 5) and original texts.
    - If total text is too large (>100k chars), batch positions into groups of 20-30 and send multiple calls.
    - Call Claude Opus via messages.parse() with ExtractionResult output_format.
    - 3x retry with exponential backoff. On total failure: return positions unchanged with warning.
    - Merge Pass 3 corrections back: Pass 3 results win over existing (pass_priority=3).
    - Tag fields with FieldSource konfidenz from Pass 3 response.

    Create `backend/v2/extraction/pipeline.py`:
    - `async def run_extraction_pipeline(parse_results: list[ParseResult], tender_id: str) -> ExtractionResult`
    - Sort files: XLSX first, then PDF, then DOCX (per user decision).
    - For each file in order:
      1. Pass 1: `extract_structural(pr)` -> `merge_positions(all_positions, pass1_results, pass_priority=1)`
      2. Pass 2: `await extract_semantic(pr, existing_positions=all_positions)` -> `merge_positions(all_positions, pass2_results, pass_priority=2)`
    - After all files: Pass 3: `await validate_and_enrich(all_positions, original_texts, source_files)`
    - Build and return ExtractionResult with positionen, summary, and collected warnings.
    - Helper: `_sort_by_format(results: list[ParseResult]) -> list[ParseResult]` with priority xlsx=0, pdf=1, docx=2, other=3.

    Update `backend/v2/extraction/__init__.py` to also export `extract_semantic`, `validate_and_enrich`, `run_extraction_pipeline`.

    Create `backend/tests/test_v2_pipeline.py`:
    - Mock anthropic client (patch `anthropic.AsyncAnthropic` or `anthropic.Anthropic`) to avoid real API calls.
    - test_pipeline_file_ordering: Verify XLSX files are processed before PDF before DOCX.
    - test_pipeline_runs_all_passes: Verify Pass 1, Pass 2, Pass 3 are all called in order.
    - test_pipeline_dedup_between_passes: Verify merge_positions called after each pass.
    - test_pass2_chunks_large_document: Verify chunking is applied for documents with page_count > 30.
    - test_pass2_retry_on_failure: Verify 3x retry on API error.
    - test_pass2_skips_failed_chunk: After 3 failures, chunk is skipped and warning added.
    - test_pass3_batches_large_position_sets: Verify Pass 3 batches when positions > 30.
    - test_pipeline_returns_extraction_result: Output is a valid ExtractionResult with positionen list.

    IMPORTANT: All tests must mock the Anthropic client. No real API calls in tests.
