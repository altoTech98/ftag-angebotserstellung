1. Create cross_doc_matcher.py:
   - SAFETY_FIELDS constant: {"brandschutz_klasse", "brandschutz_freitext", "rauchschutz", "rauchschutz_freitext", "einbruchschutz_klasse"}
   - MAJOR_FIELDS constant: {"breite_mm", "hoehe_mm", "wandstaerke_mm", "material_blatt", "material_zarge", "schallschutz_klasse", "schallschutz_db", ...all dimension/material/spec fields}
   - _normalize_position_id(id_str) -> normalized string. Strip prefixes ("Tuer ", "Pos. ", "Element ", "T-", "Nr. "), extract numeric part with regex.
   - _classify_severity(field_name) -> ConflictSeverity: CRITICAL if in SAFETY_FIELDS, MAJOR if in MAJOR_FIELDS, else MINOR.
   - match_positions_across_docs(positions_by_doc: dict[str, list[ExtractedDoorPosition]]) -> list[CrossDocMatch]:
     * For each document pair: compare all positions
     * Tier 1: Exact positions_nr match -> CrossDocMatch(confidence=1.0, method="exact_id", auto_merge=True)
     * Tier 2: Normalized ID match -> CrossDocMatch(confidence=0.92, method="normalized_id", auto_merge=True)
     * Tier 3: Same raum_nr + geschoss + oeffnungsart -> CrossDocMatch(confidence=0.7, method="room_floor_type", auto_merge=False)
     * For AI semantic matching (ambiguous cases), use asyncio.to_thread + messages.parse() with CROSSDOC_MATCHING prompts. Batch at 25 position pairs per call.
   - Export: match_positions_across_docs, _normalize_position_id (for testing), _classify_severity (for testing)

2. Create enrichment.py:
   - enrich_positions(matches: list[CrossDocMatch], all_positions: list[ExtractedDoorPosition], general_specs: list[GeneralSpec] | None = None) -> tuple[list[ExtractedDoorPosition], EnrichmentReport]:
     * For each auto_merge match (confidence >= 0.9): merge fields from both positions
     * Gap fill: If field is None in primary, copy from secondary with enrichment_source=secondary_doc, enrichment_type="gap_fill"
     * Confidence upgrade: If field konfidenz < 0.7 and other doc has konfidenz >= 0.7, replace with enrichment_type="confidence_upgrade"
     * NEVER overwrite field with higher konfidenz (anti-pattern from research)
     * Apply general specs: For each GeneralSpec, find positions matching the scope (e.g., geschoss=="OG"), apply affected_fields to EMPTY fields only with konfidenz=0.7, enrichment_type="general_spec"
     * Track all enrichment in FieldSource with enrichment_source and enrichment_type
     * Build EnrichmentReport with per-document stats and human-readable zusammenfassung
   - Export: enrich_positions

3. Create conflict_detector.py:
   - detect_and_resolve_conflicts(matches: list[CrossDocMatch], all_positions: list[ExtractedDoorPosition], client: Optional[anthropic.Anthropic] = None) -> list[FieldConflict]:
     * For each auto_merge match: iterate fields, find where both positions have non-None different values
     * Severity from _classify_severity (deterministic, no AI needed)
     * For resolution: batch conflicts and send to Claude via asyncio.to_thread + messages.parse() with CROSSDOC_CONFLICT prompts
     * AI determines which value is correct based on doc type priority (PDF spec > XLSX list for technical details), context, and domain logic
     * Store BOTH values in FieldConflict. resolution = chosen value, resolution_reason = AI reasoning, resolved_by = "ai"
     * For string comparison: compare .value for enums, str() for others. Handle None vs non-None (that's enrichment, not conflict).
     * Use 3x retry with exponential backoff (same pattern as pass3_validation.py)
   - Export: detect_and_resolve_conflicts
