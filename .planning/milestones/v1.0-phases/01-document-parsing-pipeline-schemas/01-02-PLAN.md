**PDF Parser (`backend/v2/parsers/pdf_parser.py`):**

Port v1's document_parser.py fallback chain into a clean `parse_pdf(content: bytes, filename: str = "") -> ParseResult` function:

1. **Primary: pymupdf4llm** — `pymupdf4llm.to_markdown(content)` with `table_strategy='lines_strict'`. If text length > 100 chars, use this result. Per RESEARCH pitfall 1: if table count seems low on pages with suspected tables, retry with `table_strategy='text'`.

2. **Fallback: PyMuPDF text + pdfplumber tables** — Use `fitz.open(stream=content)` for page text, `pdfplumber.open(io.BytesIO(content))` for table extraction. Combine into markdown format.

3. **Last resort: OCR** — If both above produce < 100 chars of text, try pytesseract. Port v1's OCR configuration (tesseract binary path detection on Windows, German+English language). Use `fitz` to render pages to images, then pytesseract on each image. If pytesseract is not installed, log warning and return what we have.

4. **Error handling**: Wrap entire function in try/except. On any exception, return ParseResult with empty text and warning message. Never raise — per user decision "Weitermachen + warnen". Specifically catch encrypted PDF errors from fitz.

5. Record which method succeeded in `metadata["method"]`. Set `page_count` from fitz doc. Populate `tables` list with any extracted table markdown.

**DOCX Parser (`backend/v2/parsers/docx_parser.py`):**

Create `parse_docx(content: bytes, filename: str = "") -> ParseResult`:

1. Open with `python-docx`: `Document(io.BytesIO(content))`
2. Extract paragraphs with style context: prefix headings with `# `, `## `, etc. based on style name containing "Heading"
3. Extract tables: for each table, build markdown table format with `|` delimiters. Add each table markdown to `tables` list.
4. Combine paragraphs and tables into full text (tables inline where they appear, or appended after paragraphs)
5. Set metadata["styles_found"] to list of unique style names found
6. Error handling: wrap in try/except, return ParseResult with warning on failure, never raise

Both parsers: use structured logging (`logger = logging.getLogger(__name__)`), import ParseResult from `v2.parsers.base`, import ParseError from `v2.exceptions` (use for internal logic only, always catch before returning).

5. **`match_columns(headers: list[str]) -> dict[str, str]`** — Map each header to best matching known field using SequenceMatcher. Return dict of {header_text: canonical_field_name}. Only match if similarity > 0.65 (per v1 logic). Store unmatched columns too.

6. **Main `parse_xlsx` function:**

   - Populate `tables` list with markdown table of each sheet's data
