## Critical Gaps (3)

### 1. ANLZ-04: Analysis Execution Flow — BROKEN

**Severity:** Critical — blocks core product functionality
**Location:** `frontend/src/lib/actions/analysis-actions.ts` → Python backend

The file bridge from Vercel Blob to Python (`prepareFilesForPython`) has two issues:
- Calls `POST /api/upload/project` — this endpoint does not exist in Python (only `/upload` and `/upload/folder`)
- Sends `X-API-Key` header instead of `X-Service-Key` — service_key_middleware rejects with 401

**Impact:** The entire analysis flow (file upload → AI matching → results → Excel) cannot execute end-to-end. This blocks ANLZ-04 directly and makes ANLZ-05, RSLT-01..04 unreachable.

**Fix:** Update endpoint to match Python's existing `/upload/folder` route and change header to `X-Service-Key`.

### 2. DASH-03: Dashboard Statistics — WRONG DATA

**Severity:** Medium — dashboard shows zeros instead of real data
**Location:** `frontend/src/lib/actions/dashboard-actions.ts` line 93

`getMatchGapStatistics` reads `result.matched_items` but `AnalysisResult` type stores the array under `result.matched`. Statistics widget always shows 0 matches, 0 gaps, 0% confidence.

**Fix:** Change `matched_items` to `matched` (and similarly for gap counting).

### 3. INFRA-05: Analysis Completion Email — WRONG DATA

**Severity:** Low — email sends but with incorrect stats
**Location:** `frontend/src/lib/actions/analysis-actions.ts` lines 162-163

`sendAnalysisCompleteEmail` reads `result.match_items` and `result.gap_items` but stored result uses `result.matched` and `result.unmatched`. Email always reports 0/0.

**Fix:** Change JSON key access to match `AnalysisResult` type definition.

## Tech Debt (3 items)

### Phase 12: RBAC Pattern Bypass
`archiveProject`, `deleteProject`, `shareProject`, `removeShare` use raw `session.user.role` string comparison instead of `auth.api.userHasPermission()`. Works but bypasses the declarative RBAC system.

### Phase 15: Hardcoded Email Sender
`EMAIL_FROM` in `email.ts` is hardcoded. Should read from `RESEND_FROM_EMAIL` env var.

### Phase 13: Missing Cancel Endpoint
`handleCancelConfirm` calls a non-existent `POST /analyze/cancel/{id}` — silently fails, Python background thread continues.
