**catalog-upload.tsx (client component):**
Create a drag-and-drop upload component for Excel/CSV catalog files. Follow the FileDropzone pattern from Phase 12-02 (native HTML5 drag-and-drop, no library):
- Accept `.xlsx` and `.csv` files only
- On drop/select: create FormData, call `uploadCatalog` server action
- Show uploading state with a progress indicator
- Display validation results after upload: total products count, main products, categories, and any errors/warnings
- Errors shown in red list, warnings in yellow list
- Success shows green confirmation with product stats
- Use sonner toast for quick feedback, inline display for detailed validation results
- Permission: only show upload button if user has `catalog.upload` permission (pass `canUpload` prop from server page)

**catalog-stats.tsx (client component):**
Simple stats cards showing for a catalog: total products, main products, accessory products, categories count. Takes stats data as props.

- Each catalog card shows: name, description, active version info (product count, upload date, file name), link to detail page `/katalog/[catalogId]`
- If no catalogs exist, show an empty state encouraging upload
- Link to version history: each card has a "Versionen" button linking to `/katalog/[catalogId]/versions`

**catalog-table.tsx (client component):**
Product data table with search and filtering:
- Props: `catalogId: string`, `initialProducts`, `categories: string[]`, `canEdit: boolean`
- Search input at top: debounced text search (300ms) that calls `getCatalogProducts` with search param
- Category filter dropdown: shadcn Select with all category names, "Alle Kategorien" as default
- Product table with columns: Kategorie, Kostentraeger (product key), Tuertyp, Brandschutz, Masse, and an Actions column
- Pagination at bottom: page number, total count, prev/next buttons
- Each row has an "Bearbeiten" (edit) button if `canEdit` is true, opening ProductEditDialog
- Each row has a "Loeschen" (delete) button if `canEdit` is true, with confirmation
- Products with overrides show a small "Bearbeitet" badge
- Use `useTransition` for loading states during search/filter/pagination

**product-edit-dialog.tsx (client component):**
Dialog form for editing product fields:
- Props: `catalogId: string`, `product?: ProductData`, `mode: 'add' | 'edit'`, `onClose: () => void`
- Form fields (the ~15 key fields from research): Kategorie, Kostentraeger, Tuertyp, Tuerblattausfuehrung, Anzahl Fluegel, Brandschutzklasse, Lichtmass max, Schallschutz (dB), Widerstandsklasse, Glasausschnitt, Oberflaeche
- Use shadcn Dialog, Input, Select components
- On submit: call `saveProductOverride` with action "edit" or "add" and the form data as JSON
- Show loading state during save, toast on success/error
- For delete: separate confirmation dialog, calls `saveProductOverride` with action "delete"
