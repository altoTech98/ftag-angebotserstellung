       b. `saveAnalysisResult(analysisId: string, result: unknown)`: auth check, update Analysis record with status "completed", result JSON, endedAt = new Date(), revalidatePath for project page. Return { success: true }. On error return { error: string }.

    2. Create `step-progress.tsx`:
       - Props: `jobId: string | null`, `onComplete: (result: AnalysisResult) => void`, `onFailed: (error: string) => void`, `onCancel: () => void`
       - On mount (when jobId is set), call `connectToAnalysis(jobId, onEvent, onError)`
       - Clean up SSE connection on unmount
       - Map progress text to stages using ANALYSIS_STAGES patterns from types.ts:
         - For each stage: show CheckCircle2 icon (done), filled circle (active), empty circle (pending)
         - Extract counter from progress text (regex for "(\d+)/(\d+)")
         - Show counter next to active stage label
       - Full-width shadcn Progress bar: compute percentage from activeStage (0=10%, 1=35%, 2=60%, 3=85%, completed=100%)
       - Show "Analyse laeuft..." text with spinner
       - Cancel button ("Analyse abbrechen"): onClick shows a confirmation Dialog ("Analyse wirklich abbrechen?"), on confirm calls `fetch('/api/backend/analyze/cancel/${jobId}', { method: 'POST' })` then calls onCancel
       - When SSE delivers status=completed: extract result data, call onComplete(result)
       - When SSE delivers status=failed: call onFailed(error message)

    3. Update `analyse/client.tsx` to replace step 4 and step 5 placeholders:
       - Step 4: Render StepProgress component
       - Wire the "Weiter" button on step 3 to:
         a. Call `createAnalysis(project.id)` server action to get analysisId
         b. Call `fetch('/api/backend/analyze/project', { method: 'POST', body: JSON.stringify({ project_id: project.id }) })` to start analysis and get job_id
         c. Dispatch START_ANALYSIS with jobId and analysisId
         d. Auto-advance to step 4
       - Wire StepProgress onComplete:
         a. Call `saveAnalysisResult(analysisId, result)` server action
         b. Dispatch SET_RESULT with result
         c. Show "Analyse abgeschlossen" toast (sonner)
         d. After 1.5s, auto-advance to step 5
       - Wire StepProgress onFailed:
         a. Dispatch ANALYSIS_FAILED with error
         b. Show error toast (sonner)
         c. Navigate back to step 3
       - Wire StepProgress onCancel: dispatch ANALYSIS_FAILED, go to step 3
