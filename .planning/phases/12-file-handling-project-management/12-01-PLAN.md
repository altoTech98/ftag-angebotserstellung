   - Export async POST handler
   - Authenticate via `auth.api.getSession({ headers: await headers() })`
   - Return 401 JSON if no session
   - Use `handleUpload` from `@vercel/blob/client` with:
     - `allowedContentTypes`: PDF, DOCX, XLSX MIME types
     - `addRandomSuffix: true` (prevent filename collisions)
     - `tokenPayload`: JSON.stringify({ userId: session.user.id })
     - `onBeforeGenerateToken`: return config above
     - `onUploadCompleted`: empty (not reliable locally per research)
   - Return JSON response from handleUpload

3. Create `frontend/src/lib/actions/project-actions.ts`:
   - 'use server' directive
   - `createProject(formData: FormData)`: authenticate, check permission (project:create), prisma.project.create with name (required), customer, deadline, description, ownerId. revalidatePath('/projekte'). Return project.
   - `archiveProject(projectId: string)`: authenticate, verify ownership or admin, prisma.project.update status to "archived". revalidatePath('/projekte').
   - `deleteProject(projectId: string)`: authenticate, verify ownership or admin. Query all file blobUrls for the project, call `del(urls)` from @vercel/blob to clean up blobs, then prisma.project.delete (cascades File/Analysis/ProjectShare). revalidatePath('/projekte').

4. Create `frontend/src/lib/actions/file-actions.ts`:
   - 'use server' directive
   - `saveFileMetadata({ name, blobUrl, downloadUrl, size, contentType, projectId }: FileMetadataInput)`: authenticate, prisma.file.create with uploadedBy = session.user.id. revalidatePath for project detail page.
   - `deleteFile(fileId: string)`: authenticate, find file, call `del(file.blobUrl)` from @vercel/blob, prisma.file.delete. revalidatePath.
