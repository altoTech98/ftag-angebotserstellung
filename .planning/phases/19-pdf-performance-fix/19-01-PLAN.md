<objective>
Replace pdfplumber text extraction with PyMuPDF for 10-50x speed improvement, fix the max_pages=30 cap, add per-page progress logging, and verify pydantic compatibility at startup.

Purpose: The analysis pipeline fails on real FTAG tender documents (286 pages) because pdfplumber text extraction takes ~10 minutes. PyMuPDF (C-based MuPDF engine) completes the same work in seconds. This is the root cause blocking the entire v2.1 analysis pipeline.

Output: Modified document_parser.py with hybrid parsing (PyMuPDF text + pdfplumber tables), PyMuPDF in requirements.txt, pydantic version check in main.py startup, updated tests.
</objective>
