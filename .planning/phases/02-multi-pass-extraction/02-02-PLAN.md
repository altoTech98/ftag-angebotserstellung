    - test_upload_multiple_files: POST /api/v2/upload with 3 files returns all file metadata and total_files=3
    - test_upload_append_to_tender: POST /api/v2/upload with existing tender_id adds files to that tender, total_files reflects cumulative count

    Create `backend/v2/routers/__init__.py` as empty package init.

    Create `backend/v2/routers/upload_v2.py`:
    - Router with `prefix="/api/v2"` and `tags=["V2 Upload"]`
    - In-memory tender storage: `_tenders: dict[str, dict]` mapping tender_id to `{"files": [...], "status": "uploading", "created_at": datetime}`
    - `POST /upload`: Accept `files: list[UploadFile]` and optional `tender_id: str = Form(None)`.

      - If tender_id provided but not in _tenders: return 404

    - `GET /tender/{tender_id}`: Return tender status with file list
    - Do NOT use `Form(None)` for tender_id with multipart -- use `Query(None)` or URL parameter instead, since multipart with File and Form can be tricky. Use query parameter: `tender_id: Optional[str] = None` as query param.

    Write tests using FastAPI TestClient. Use `conftest_v2.py` fixtures for sample file bytes.
    Tests first (RED), then implement (GREEN).
