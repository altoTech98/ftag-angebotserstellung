    Create `backend/v2/extraction/pass1_structural.py`:
    - `extract_structural(parse_result: ParseResult) -> list[ExtractedDoorPosition]`
    - For XLSX: Parse the tabular text line by line. The XLSX parser's text output is formatted with columns matched to fields. Use regex patterns from KNOWN_FIELD_PATTERNS to identify position rows. Extract position number via POSITION_NR_PATTERNS (patterns: `\d{1,3}\.\d{1,3}`, `T\d{1,3}\.\d{1,3}`, `Pos\.?\s*\d{1,3}\.\d{1,3}`). Extract dimensions by matching patterns like `\d{3,4}` near dimension column labels. Extract fire/sound class by matching enum values (EI30, EI60, Rw 37dB, etc).

    - Import KNOWN_FIELD_PATTERNS from `v2.parsers.xlsx_parser` for column recognition.

    - `chunk_by_pages(text: str, page_count: int, chunk_size: int = 30, overlap: int = 5) -> list[dict]`
    - Each chunk dict has: text, start_page, end_page
    - Split on page markers: form feeds (`\f`), `--- Page X ---` patterns, or `\n\n` as fallback
    - If page_count <= chunk_size, return single chunk
    - Handle edge cases: empty text, page_count=0, no recognizable page markers

  <action>
    Create `backend/v2/extraction/dedup.py`:
    - `merge_positions(existing: list[ExtractedDoorPosition], new_positions: list[ExtractedDoorPosition], pass_priority: int = 1) -> list[ExtractedDoorPosition]`
    - Two-phase approach:
      1. Pre-filter: Exact positions_nr match -> direct merge (no AI needed)
      2. For remaining unmatched new positions: add them as-is (AI clustering will be called by the pipeline orchestrator separately for ambiguous cases, using `ai_dedup_cluster`)
    - `ai_dedup_cluster(positions: list[ExtractedDoorPosition], anthropic_client) -> list[list[int]]` — async function that sends position summaries to Claude for clustering. Returns groups of indices that represent the same door. Uses DEDUP_PROMPT_TEMPLATE.
    - Merge logic: For each field, if new position's field is not None and pass_priority is higher, new value wins. If new field is None, keep existing. Always update quellen dict with winning value's source.
    - Conflict resolution: later pass wins (pass_priority parameter). Original value's source preserved in quellen history.

    Create `backend/v2/extraction/prompts.py`:
    - `PASS2_SYSTEM_PROMPT`: German-language system prompt for semantic extraction. Include full field list from ExtractedDoorPosition. Instruct to extract ALL positions, set low confidence when uncertain, prefer over-extraction. Use "Du bist ein Experte fuer die Analyse von Ausschreibungen fuer Tueren und Zargen."
    - `PASS2_USER_TEMPLATE`: Template with {chunk_text} and {existing_positions_json} placeholders. Tell Claude what positions were already found so it can focus on missed ones.
    - `PASS3_SYSTEM_PROMPT`: German-language prompt for cross-reference validation. Instruct to check for: missing positions, incomplete fields, wrong values, confused positions, overlooked details. Return corrected/enriched ExtractionResult.
    - `PASS3_USER_TEMPLATE`: Template with {all_positions_json} and {original_texts} placeholders.
    - `DEDUP_PROMPT_TEMPLATE`: Template with {positions_json} placeholder. Ask Claude to group positions that represent the same door. Return list of groups.
    - All prompts in German (matching domain language).
