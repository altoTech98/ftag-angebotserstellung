<objective>
Create the Next.js BFF proxy layer that forwards all /api/backend/* requests to Python with service authentication, plus SSE token issuance and a client-side SSE utility with polling fallback.

Purpose: The browser never calls Python directly for CRUD operations. All requests go through Next.js API routes which add the service key and user context headers. For SSE streaming (analysis progress), a separate token mechanism allows direct browser-to-Python connections.

Output: Catch-all proxy route, SSE token endpoint, health check endpoint, SSE client utility, proxy helper module, updated .env.example, passing tests.
</objective>

<!-- Contract with Python backend (Plan 01) -->
Python expects these headers on /api/* requests:
- X-Service-Key: must match PYTHON_SERVICE_KEY env var
- X-User-Id: Better Auth user.id
- X-User-Role: user.role (viewer/analyst/manager/admin)
- X-User-Email: user.email

Python SSE endpoint: /api/analyze/stream/{job_id}?token=xxx
- Token format: {base64url_payload}.{base64url_hmac_signature}
- Shared secret: SSE_TOKEN_SECRET env var

Python health: GET /health returns JSON { status: "healthy"|"degraded"|"error", ... }

    1. Create frontend/src/lib/python-proxy.ts:
       - Export proxyToPython(request: Request, pathSegments: string[]): Promise<Response>
       - Read PYTHON_BACKEND_URL and PYTHON_SERVICE_KEY from process.env
       - Get session via auth.api.getSession({ headers: await headers() })
       - Return 401 if no session
       - Build target URL: new URL(`/api/${pathSegments.join("/")}`, PYTHON_URL)
       - Forward query params from incoming request
       - Determine timeout: 300s for paths starting with /api/analyze, 30s for rest
       - Build headers: X-Service-Key, X-User-Id, X-User-Role (default "viewer"), X-User-Email, Content-Type
       - Use AbortController + setTimeout for timeout
       - Forward body via request.arrayBuffer() for non-GET/HEAD methods
       - Pass through Python response (status, statusText, Content-Type, body)
       - Catch AbortError -> 504, other errors -> 502
       - Always clearTimeout in finally

    2. Create frontend/src/app/api/backend/[...path]/route.ts:
       - Import proxyToPython from @/lib/python-proxy
       - Export GET, POST, PUT, DELETE handlers
       - Each handler: extract params (await ctx.params), call proxyToPython(req, params.path)
       - Follow Next.js 16 pattern: ctx.params is a Promise

    3. Create frontend/src/app/api/backend/sse-token/route.ts:
       - Export GET handler
       - Get session via auth.api.getSession({ headers: await headers() })
       - Return 401 if no session
       - Build payload: { sub: user.id, role: user.role || "viewer", email: user.email, exp: now + 600, iat: now }
       - Sign with HMAC-SHA256 using SSE_TOKEN_SECRET (from process.env, fallback to PYTHON_SERVICE_KEY)
       - Format: base64url(payload) + "." + base64url(hmac_signature)
       - Return JSON: { token, expires_in: 600 }

    4. Create frontend/src/app/api/backend/health/route.ts:
       - Export GET handler (no auth required for health check)
       - Fetch PYTHON_BACKEND_URL/health with 5s timeout
       - Return { status: "connected", python: data } on success
       - Return { status: "disconnected" } with 503 on failure

       - Add SSE_TOKEN_SECRET=dev-sse-secret-change-in-production
