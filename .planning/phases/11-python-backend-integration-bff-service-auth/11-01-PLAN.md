<objective>
Replace Python's JWT auth middleware with service key authentication and add SSE token validation for direct browser-to-Python streaming connections.

Purpose: Python backend trusts Next.js as the auth gateway. Instead of per-user JWT login, Python validates a shared service key (X-Service-Key header) on every request from Next.js, and extracts user context from X-User-Id, X-User-Role, X-User-Email headers. SSE streaming endpoints use short-lived HMAC-signed tokens passed via query parameter.

Output: Service key middleware, SSE token validator, updated CORS config, updated analyze.py SSE endpoint, passing tests.
</objective>

<interfaces>
<!-- Contracts this plan implements. Next.js (Plan 02) will send these headers. -->

Service Key Header Contract:
- X-Service-Key: string (matches PYTHON_SERVICE_KEY env var)
- X-User-Id: string (Better Auth user ID)
- X-User-Role: string (one of: viewer, analyst, manager, admin)
- X-User-Email: string (user's email address)

SSE Token Contract (HMAC-signed, base64url):
- Format: {payload_b64}.{signature_b64}
- Payload: { sub: string, role: string, email: string, exp: number, iat: number }
- Secret: SSE_TOKEN_SECRET env var (shared between Next.js and Python)
- Expiry: 10 minutes
- Passed as: ?token=xxx query parameter on SSE endpoints

Whitelist (no auth required):
- /health, /api/health, /docs, /redoc, /openapi.json, /info, /
- OPTIONS preflight requests
- /api/analyze/stream/* (uses SSE token auth instead)
</interfaces>
