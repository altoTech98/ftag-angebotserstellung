Output: Compatible token format across both systems, verified by a cross-system test.


Token format contract: base64url(JSON_payload).hex(HMAC-SHA256(JSON_payload_bytes))
- Payload part: base64url-encoded raw JSON string (both sides agree)
- Signature part: hex-encoded HMAC-SHA256 of the raw JSON bytes (Python expects this, Next.js must match)
- Secret: SSE_TOKEN_SECRET env var (with PYTHON_SERVICE_KEY fallback)
- HMAC input: the raw JSON payload bytes (NOT the base64url-encoded string)

IMPORTANT: Python signs `payload_bytes` which is `base64.urlsafe_b64decode(payload_b64)` -- i.e., the raw JSON bytes AFTER decoding from base64url. Next.js currently signs `payloadStr` (raw JSON) BEFORE base64url encoding. These produce the same bytes, so the HMAC input already matches. Only the signature output encoding needs to change from base64url to hex.
